# 🛡️ DevSecOps Multi-Scanner Tool
Универсальный Docker-контейнер для комплексного сканирования безопасности кода. Инструмент собирает данные из множества сканеров, формирует единый HTML-отчет и опционально отправляет результаты в DefectDojo.

## 🚀 Возможности
Гибкий выбор сканеров: Запуск только нужных тулзов (*Semgrep, Trivy, Bandit, Gitleaks, Gosec и др.*).

- __Консолидированный отчет__: Сборка всех находок в один автономный *HTML*-файл.

- __Интеграция с DefectDojo__: Опциональная загрузка с автоматической привязкой *HTML*-отчета.

- __Quality Gate__: Настраиваемый порог прерывания *CI/CD* (Fail on High/Medium).

## 🛠️ Как это работает (Docker)
Для запуска не нужен Compose, достаточно собрать образ и запустить его, пробросив нужные папки.

1. Сборка образа
```docker
docker build --no-cache -t {DOCKER_NAME} .
```  
2. Запуск сканирования
При запуске необходимо примонтировать две папки:

*/src* — папка с вашим исходным кодом для анализа.

*/scanner/reports* — папка, куда сохранится итоговый full_report.html и другие репорты.

так же необходимо:
```bash
mkdir reports
sudo chown 10001:10001 reports
```
после сборки можете запускать.
```docker
docker run --rm \
  -v "$(pwd)/my-project:/src" \
  -v "$(pwd)/reports:/scanner/reports" \
  (-e TOOL={TOOL/S_NAME})OPTIONAL
  {DOCKER_NAME}
```
## ⚙️ Варианты конфигурации
Выбор инструментов (Интерактивность)
Инструмент проверяет наличие файлов в папке */src* и запускает соответствующие сканеры автоматически. Вы можете управлять списком запусков через переменные окружения или просто ограничив содержимое */src*.

Интеграция с *DefectDojo* (Опционально)
Если вы хотите отправить результаты в Dojo, передайте токены при запуске. Если они не переданы, скрипт просто создаст локальный отчет.
```docker
docker run --rm \
  -v "${PWD}/{TARGET_NAME}:/scan-target:ro \
  -v "$(pwd)/reports:/scanner/reports" \
  -e DOJO_URL="{URL}" \
  -e DOJO_TOKEN="{YOUR_KEY}" \
  -e ENGAGEMENT_ID="{ID}" \
  {DOCKER_NAME}
```
## 📂 Структура проекта
__entrypoint.sh__ — логика управления сканерами и загрузкой в Dojo.

__generate_report.py__ — движок консолидации отчетов в HTML.

__Dockerfile__ — среда со всеми предустановленными утилитами (Semgrep, Gitleaks и т.д.).

__/reports__ — папка с результатами (создается автоматически, но нужна передача прав докер образу, чтобы он мог записывать отчеты).

## 🛡️ Quality Gate (Статус сборки)
Контейнер вернет:

__Exit Code 0__: Если уязвимости не найдены или их уровень ниже установленного порога.

__Exit Code 1__: Если найдены критические уязвимости (*Critical/High/Medium*), требующие немедленного исправления.
